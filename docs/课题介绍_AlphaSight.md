# AlphaSight 实证洞察与证伪挑战赛

## 课题简介

根据一年的美股公司**财报 / 研报 / 新闻 / 社交媒体 / 行情**多源语料和一个 OpenAI 兼容 LLM 端点，**离线**实现一个能"写研报"也能"审研报"的 Agent，比拼**事实可追溯的洞察力**与**证伪能力**。

## 课题亮点

- **业界真实需求实践**：50 支美股大盘股、完整 1 年（2025-01-01 ~ 2025-12-31）的 SEC 备案（10-K/10-Q/8-K）、新闻、推特、结构化研报数据、OHLCV 行情。**题型覆盖广**：6 个固定题型（财报解读 / 行业比较 / 事件影响 / 风险归因 / 市场反应 / 叙事反转）+ 4 个自由命题，单篇上限 1000 字，引用不计字数。
- **现成代码框架**：仓库提供完整的代码框架。**你专注做"怎么做更好"，不用在脚手架上耗时间。**


## 所需基础

### 必备
- **Python**
- **LLM API 基本概念**：System / User message、temperature、JSON 模式输出、上下文窗口。
- **基础检索思路**：至少一种——关键词 / TF-IDF / BM25 / Embedding 向量检索。
- **英文阅读能力**：语料**全英**（10-K、新闻、推特），写出来的研报也建议英文。

### 加分项
- 向量库 / RAG 实战（FAISS、Chroma、LanceDB 任选）。
- vLLM / Ollama 自托管开源模型经验（Qwen / DeepSeek / Llama）。
- Agent 框架：多轮工具调用、自我校验、planner-executor。
- 后训练：SFT / LoRA / DPO，能在小算力上 finetune。
- 金融基础阅读：能看懂 10-K 的 Item 1A / Item 7 / Item 8 在讲什么。


## 研究 / 实践目标

### 工程层
1. **构建一套自动化的"知识库整理 Agent"**：把 filings / news / research / social / prices 五类异构语料抽取、清洗、切块、索引成可被 LLM 高效检索的形式。
2. **生成 Agent**：能根据题目主题在本地知识库内做**多跳证据收集 → 综合推理 → 引用规范**的研报生成。
3. **审核 Agent**：能给陌生报告按 4 维 rubric（`factual_support` / `insight_depth` / `coverage` / `readability`）打分，并指出**有据可查**的逻辑漏洞、引用错误、未支撑论断。

### 研究层
- **如何让模型在语料中"挖到非显然的洞察"而非复读披露原文？**
- **如何让 Reviewer Agent 真正具有专业评判能力**。
- 跨源叙事：怎么把 10-K 的财务披露、8-K 的事件、新闻反应、推特情绪、价格走势串成一条**可证伪**的因果链？进而做到预测未来？

## 预期成果

- 一份可重现、可离线运行的代码脚本。
- 10 份在固定 6 题 + 自由 4 题上的研究报告（盲测数据上现场生成）。
- 主观答辩 PPT：问题定义、检索 / Agent 架构、关键工程权衡、消融对比。

## 评分组成（来自赛事方）

| 项目 | 权重 |
|---|---|
| 评委对你 10 份报告打分 | **20%** |
| Reviewer Agent 在他队报告上"挑出真问题"的得分排序 | **20%** |
| 主观答辩（问题分析 / 工程量 / 创新性） | **60%** |

## 项目特色

- **"写得好"和"审得准"同等重要**：单优一头打不到第一。
- **完整工程闭环**：从数据预处理 → 索引 → 检索 → 生成 → 自审 → 互审 → 离线打包，端到端走一遍真实 AI 工程项目的所有环节。
